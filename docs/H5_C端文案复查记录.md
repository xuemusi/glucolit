# H5 C 端文案复查记录

日期：2026-07-04

## 目标

扫描当前 GLUCOLIT H5 页面中不适合直接面向 C 端用户展示的文案、字段和内容组织方式，优先消除开发后台话术、Demo 痕迹、医学术语裸露和用户无法理解的指标堆叠。

## 复查结论

当前 H5 的合规边界整体谨慎，核心问题不在医疗风险，而在表达层仍混有产品、研发和后台数据语言。典型问题是“后续要沉淀的字段”：这类说法适合需求文档和数据建模，不适合直接展示给用户。用户看到后无法判断自己要做什么，也容易产生“还要补很多复杂检查”的压力。

## 本轮修改范围

1. 登录页、首页、AI 工具页、行动页、“我的”页和科普页的页面文案。
2. 健康档案种子数据中直接展示给用户的说明、标签和指标分组。
3. 内容卡片中“原站”相关来源标签。

## 暂不处理

1. 医学指标本身不删除，只调整展示标题和说明层级。
2. 不改底层数据结构字段名，不影响 API 和数据库。
3. 不改 AI 输出 Prompt 的医学规则，只改前端展示文案。
4. 不做复杂折叠交互，本轮先做文案替换。

## 推荐后续检查

1. 在手机尺寸下检查“我的”页专业指标标签是否仍然过密。
2. 检查报告分析结果页是否需要给 HbA1c、OGTT、HOMA-IR 增加解释入口。
3. 科普页后续可按“先看什么、怎么做、何时复查”重组内容，而不是只列研究卡片。
