## 1. 结论

这份文档不是单次报告解读，而是一份纵向健康档案。它把“报告原始指标、复查对比、计算指标、阶段判断、随访目标、每日行动”放在同一个固定格式里。产品需要新增“我的”功能，把用户已确认的固定指标沉淀下来，并让后续 AI 报告识别持续补充这个档案。

MVP 调整目标：

- 底部导航新增“我的”，承载个人健康档案。
- 在“我的”里展示固定档案、核心趋势、OGTT 曲线、风险优先级、复查计划和每日清单。
- 当前先用用户提供的健康档案做结构化种子数据；后续报告识别结果进入 `analysis_results` 后，再逐步抽取到长期指标表。
- 医疗表达保持安全边界：界面使用“缓解、改善、趋势观察、医生确认”，不把产品描述成诊断、治疗或治愈工具。

### 2.3 OGTT 判读规则与形态

- 7.8 mmol/L 是 OGTT 2h 常用红线，不是 1h 红线。
- 经典 OGTT 口径：
  - 2h < 7.8：正常糖耐量。
  - 2h 7.8-11.0：糖耐量受损。
  - 2h >= 11.1：糖尿病阈值，需医生确认。
- 1h 新标准参考：1h >= 8.6 mmol/L 才进入中度高血糖/高风险讨论；复查 1h 为 8.05，属于正常区偏上沿。
- 曲线形态变化：
  - 基线峰值时间：2h。
  - 复查峰值时间：1h。
  - 基线峰值高度：9.42。
  - 复查峰值高度：8.05。
  - 复查 2h 已回落到 6.36，3h 回落到 4.8。

### 2.8 复盘、时间线和维护期

时间线：

- 2026-01-21：首次 OGTT，2h 血糖 9.42。
- 2026-02-21：内分泌科确诊，HbA1c 5.8%，桥本，胰岛抗体阴性。
- 2026-02 下旬：启动 90 天重养计划。
- 2026-06-17：复查验证，2h 血糖 6.36，HbA1c 5.7，HOMA-IR 1.44。

维护期核心：

- 这类结果应表达为“缓解/改善”，不是“治愈”。
- 优先盯内脏脂肪 / 肝脂、肌肉量、餐后 1h/2h 血糖、HOMA-IR、甲功。
- 早期预警顺序：内脏脂肪回升 -> HOMA-IR 抬头 -> 餐后 1h/2h 变钝 -> HbA1c 最后变化。

### 2.10 分型与机制

原文包含三层分型：

- 四阶段定位：第一阶段末期 -> 第二阶段早期。
- DZD 糖前 6 亚型：最接近 Cluster 3 脂肪肝进展型早期，Cluster 6 部分符合。
- Ahlqvist T2D 5 亚群：均不完全匹配。
- 综合画像：瘦型 TOFI 复合代谢异常，叠加 β 细胞分泌节律紊乱、异位脂肪沉积、桥本自身免疫。
- 功能分型：迟钝型为主，轻度疲劳型特征。

产品注意：这些分型适合放在“医生沟通摘要 / 专业模式”，MVP 首页不直接渲染全部术语，避免用户把产品当诊断工具。

### 2.11 个体化方案

饮食策略：

- 进餐顺序：蔬菜 -> 蛋白质 -> 主食。
- 蛋白先行：餐前 10 分钟 30g 蛋白质。
- 碳水重构：白米换糙米 + 藜麦，白面换全麦，减少白粥/烂面/糯米饭。
- 每餐定量：主食约一个拳头，按餐后血糖调整。
- 分餐制：两餐间隔超过 5 小时可加餐，如 10g 坚果 + 无糖酸奶 / 半个苹果。
- 限时进食：14:10，第一周可从 12:12 起步。

运动策略：

- 餐后 30-60 分钟快走 / 骑车 20-30 分钟。
- 每周 3 次抗阻训练，40-50 分钟。
- 避免空腹高强度运动；运动前血糖偏低时先加餐。

补充剂：

- 镁 400mg/天。
- 铬 200μg/天。
- α-硫辛酸 600mg/天。
- 维生素 D 2000IU/天，前提是检测缺乏。

产品注意：补充剂不能做默认推荐，只能作为“医生/营养师确认项”。

### 2.12 监测、目标、随访和清单

监测目标：

- 空腹：4.4-7.0 mmol/L。
- 餐后 2h：<10.0 mmol/L。
- 餐后 3h：低于餐后 2h，且 <7.8。
- 餐后 4-5h：>3.9。
- 睡前：5.6-7.8。
- 餐前后差值：<2.2。
- 日内峰谷差：<4.4。

阶段目标：

- Week 1：数据采集 + 认知建立。
- Week 2-4：餐后峰值开始下降。
- Week 5-8：空腹血糖稳定。
- Week 9-12：体重下降 3-5kg，OGTT 曲线改善。
- 1-3 个月：餐后 3h <7.8，餐前后差值 <2.2。
- 3-6 个月：体重下降 5%，HOMA-IR <1.5。
- 6-12 个月：原文写 HbA1c <7.0%，但结合上下文更可能应为 <5.7%；产品文案应标为“待用户/医生确认目标”。

复查：

- 1 个月后：复诊，评估饮食运动和血糖记录。
- 3 个月后：复查 OGTT 或空腹 + 餐后血糖、HbA1c、肝肾功能。
- 每 6 个月：眼底、尿微量白蛋白、颈动脉超声。
- 每年：C 肽释放试验。

每日执行清单：

- 进餐顺序：蔬菜 -> 肉/蛋/奶/豆 -> 主食。
- 餐前 10 分钟蛋白质。
- 餐后 30-60 分钟快走 20 分钟。
- 重点测餐后 3h 和睡前。
- 每周 3 次抗阻训练。
- 食物日志 + 血糖数据。
- 睡前复盘当天是否让胰岛“加班”。

CGM 复盘：

- 餐后峰值 <7.0：可复制。
- 7.0-7.8：可接受，优化配比。
- >7.8：触发拆解。
- 每周看最低 3 餐、最高 3 餐、平均血糖、体重和腰围变化。

## 3. 产品需求

### 3.1 新增“我的”页面

页面模块：

- 健康档案摘要：阶段、最近复查日期、核心状态、医疗边界。
- 指标趋势：2h 血糖、HOMA-IR、HbA1c、胰岛素峰值。
- OGTT 曲线对比：葡萄糖、胰岛素、C 肽的基线与复查。
- 固定指标档案：身体成分、甲功、血脂、肝酶、抗体、合并症。
- 复发预警：内脏脂肪/肝脂、肌肉量、餐后血糖/HOMA-IR、甲功。
- 复查计划：1 个月、3 个月、6 个月、年度。
- 我的行动清单：饮食、运动、记录、睡眠前复盘。
- 最近 AI 报告抽取：如果用户刚上传报告，展示从 `analysis_results.result_json.ogtt` 抽出的可复用指标。

### 3.2 数据模型建议

MVP：

- `healthProfile` 静态结构化数据先放入代码，来源标注为用户提供健康档案。
- `GET /api/app-state` 返回 `healthProfile`。
- 前端“我的”页面渲染该结构。
- 最近报告识别结果继续存 `analysis_results`，前端只做最近一次抽取展示。

后续 D1 表：

- `health_profiles`：用户长期健康档案摘要。
- `health_metric_snapshots`：指标快照，字段包含 `user_id`、`metric_key`、`value`、`unit`、`tested_at`、`source_analysis_id`、`source_type`、`confirmed_by_user`。
- `health_assessments`：阶段判断、风险优先级、随访计划。
- `health_action_templates`：由档案生成的个性化行动模板。

### 3.3 报告识别后的抽取规则

可从当前 `analysis_results.result_json` 抽取：

- `result.ogtt.glucose.fasting/h1/h2/h3`
- `result.ogtt.insulin_raw.values`
- `result.ogtt.c_peptide_raw.values`
- `result.ogtt.hba1c`
- `result.ogtt.derived.homa_ir`
- `result.ogtt.flags`
- `result.fields`

限制：

- OCR 没识别到的指标不能补。
- 未经用户确认的指标要标记为“待确认”。
- 单次识别结果不能覆盖长期档案，只能作为“新快照候选”。

### 3.4 合规边界

- 页面不使用“治愈”“保证”“必须”“不能吃”“建议吃药/停药”等表达。
- 原文出现“逆转”时，产品 UI 改成“缓解/改善/回到常用正常区间”。
- 分型、诊断和补充剂用“医生沟通项/随访项”表达。
- 所有报告和档案均提示：用于健康教育、趋势观察和生活方式支持，不替代医生诊断治疗。

## 4. 本次开发决策

本次先做低风险 MVP：

- 替换底部“陪伴”为“我的”，因为当前需求比电话陪伴更核心。
- 不新增 D1 migration，避免仓促引入未确认的数据写入模型。
- 新增结构化健康档案数据文件，保留来源和安全改写。
- `app-state` 返回健康档案。
- 前端新增“我的”视图，显示档案和最近报告抽取。
- 更新技术文档、测试用例和开发进度。

后续再做：

- 健康档案 D1 表。
- 报告识别后的用户确认流程。
- 从确认后的快照生成趋势图和复查提醒。
