# GLUCOLIT UI Redesign Comparison and Decisions

日期：2026-07-03  
对比对象：

- 原站：`https://glucolit.vercel.app/zh`
- 当前 MVP：`https://glucolit.xuemusi.com`

## 1. 本轮目标

先沉淀目标，再修改界面。后续每次 UI 调整必须在本文件追加记录。

目标：

1. 借鉴原站的可信内容品牌感和深色“今日关注”视觉。
2. 保留当前 MVP 的黑客松闭环：手机号进入、监控预警、AI 工具、行动打卡、陪伴、科普。
3. 重做首页首屏，尤其是当前“表格式指标 + 简陋折线图”观感。
4. 配料表分析、拍照分析、监控预警后续要从“样例按钮”升级为更像真实功能的入口与状态。
5. 不牺牲移动端 390x844 首屏可读性。

## 2. 原站观察

原站更像内容/指南站，不是 H5 工具 Demo。

首屏特征：

- 顶部品牌清晰：logo + GLUCOLIT。
- 大标题是“糖尿病前期与代谢健康干预指南”。
- 视觉更偏专业内容品牌，强调 evidence-based。
- 首屏右侧/下方有深色“今日关注”卡片，视觉重心明显。
- 页面后续是 Topic clusters：糖前基础、CGM、饮食、运动睡眠、压力情绪、代谢健康。

可借鉴：

- 品牌可信感。
- 深色重点卡片。
- “证据 -> 可执行建议”的表达。
- 分类入口用大卡片承载，不要小表格堆砌。

不直接照搬：

- 原站是长内容站，不适合直接作为黑客松 H5 工具首页。
- 原站没有注册、D1 状态、AI 上传、行动打卡闭环。

## 3. 当前 MVP 差异与问题

当前 MVP 已完成工程闭环，但 UI 问题明显：

| 项目 | 当前状态 | 问题 |
| --- | --- | --- |
| 首页首屏 | 问候卡 + 今日代谢状态 + 指标网格 | 视觉像普通表格，缺少产品记忆点 |
| 图表 | 简单 SVG 折线 | 太像占位图，不像真实健康监测 |
| 监控预警 | 首页有 AI 预警列表 | 缺少“持续监控/今日关注”的强状态表达 |
| 拍照分析 | AI 工具页只有“使用样例开始分析” | 不像拍照/上传功能 |
| 配料表分析 | 有 tab 和兜底结果 | 入口弱，用户难理解是“拍配料表” |
| 品牌感 | 有 GLUCOLIT 标识 | 没有原站那种 evidence-based 专业感 |

## 4. 调整决策

### 4.1 首页结构

采用“工具 H5 + 专业简报”混合首页：

1. 顶部保留 GLUCOLIT。
2. 首屏第一块改成深色 `今日关注` 状态卡。
3. 深色卡展示：
   - `AI 监控中`
   - 今日状态 `需关注/已响应/今日已完成`
   - 餐后峰值、睡眠、未打卡天数
   - 一句风险解释
4. 原来的表格式指标网格降级为“监测信号”横向/紧凑卡，不再作为首屏视觉中心。
5. 图表改成更像 CGM/餐后曲线的 mini trend，而不是大表格。

### 4.2 AI 工具入口

AI 工具页要显式表达“拍照/上传”：

- 报告解读：`拍报告 / 用样例报告`
- 餐盘分析：`拍餐盘 / 用样例餐盘`
- 配料表分析：`拍配料表 / 用样例配料表`

MVP 仍可使用样例结果，但视觉和文案不能像单纯按钮。

### 4.3 监控预警

首页增加“监控预警”模块语言：

- 从 `AI 预警` 改为 `AI 监控预警`
- 每条预警显示：信号、影响、行动
- 深色今日关注卡承担“监控中”的强状态表达。

### 4.4 视觉风格

使用 Stitch 重新生成糖前患者视觉方向：

- 关键词：prediabetes, metabolic health, evidence briefing, gentle coaching, action dashboard。
- 基调：专业、可信、冷静、有行动感。
- 色彩：深墨绿/深青绿色作为主视觉，薄荷绿表示完成，琥珀色表示关注，珊瑚色只用于轻风险。
- 避免：大面积表格、廉价渐变、过多边框、泛 SaaS dashboard 感。

## 5. 变更记录

### 2026-07-03 初始记录

- 建立本文件，记录原站与当前 MVP 差异。
- 决定先改首页首屏、图表、AI 工具拍照入口、监控预警表达。
- 后续代码修改必须追加本节。

### 2026-07-03 首页与拍照入口首轮调整

- 根据原站首屏和 Stitch brief，首页第一屏从浅色问候卡 + 表格式指标改为深色 `今日关注 / AI 监控中` 卡。
- 删除首页首屏的 2x2 表格化指标视觉，改为深色卡内三个信号块：餐后峰值、睡眠、未打卡。
- 图表从白底卡片中的简化折线改为深色卡内 CGM/餐后趋势 mini chart。
- `AI 预警` 改名为 `AI 监控预警`，每条预警拆成信号、影响、行动三段。
- 首页新增 `拍照分析` 三个快捷入口：报告、餐盘、配料表。
- AI 工具页从单个“使用样例开始分析”按钮改为 scanner 风格上传区域，提供 `拍照上传` 与 `使用样例` 两个入口。
- 当前拍照上传仍走稳定样例分析结果，避免黑客松演示被真实模型调用阻断；界面和流程已支持选择图片并触发对应分析。

### 2026-07-03 Tokendance 大模型接入

- 用户提供 Tokendance OpenAI-compatible 网关和 `kimi-k2.6` 模型。
- `/api/analyze` 从固定样例改为优先调用模型生成结构化 JSON；模型失败、输出不合规或密钥缺失时才回落本地样例。
- 模型密钥存入 Cloudflare Pages Secret `TOKENDANCE_API_KEY`，不写入仓库。
- 前端拍照上传会把图片转为 base64 传给 `/api/analyze`；若模型不支持图片或读取失败，后端保持兜底可用。
- 分析结果界面显示 `模型分析 · kimi-k2.6` 或 `演示兜底结果`，便于现场判断当前通路。
- 后端会对模型输出做合规归一化：配料表购买建议限定为 `更适合/需控制/建议替换`，餐盘碳水风险限定为 `低/中/偏高`，命中禁用医疗承诺词时直接回落样例。

### 2026-07-03 原站社区内容录入

- 从 `https://glucolit.vercel.app/zh` 抽取原站公开内容清单，确认共有 6 个指南专题和 9 篇研究解读。
- 科普页从 3 篇占位文章调整为“原站社区内容库”，展示指南专题与研究解读两个区块。
- `content.guides` 新增 6 个 Topic clusters；`content.articles` 扩展为 9 篇原站研究解读，保留日期、来源、标签、摘要和原站链接。
- 当前决策是不在 H5 内复刻长文详情页，先保证内容完整可见并可跳转原站阅读，避免拖大黑客松 MVP 范围。
- 录入细节沉淀到 `docs/ORIGIN_CONTENT_IMPORT.md`。

### 2026-07-03 科普内容详情页

- 用户反馈科普内容不能直接点击看详情。
- 科普卡片从单一原站链接改为 `查看详情` + `原站` 两个动作。
- `查看详情` 在当前 H5 内打开指南/文章详情视图，展示摘要、标签、学习路径或研究阅读方式、证据边界和原站完整内容入口。
- 详情页仍保持健康教育边界，不复制原站全文，不做诊断、治疗或用药建议。

### 2026-07-04 首页硬件设备接入浮窗

- 用户提出首页加入硬件设备接入浮窗，例如 Apple Watch，并希望视觉更有动效。
- 决策：不把设备接入做成首页主卡，避免抢走“今日健康概览”和主行动；改为右下角浮窗入口，展开后作为轻量接入面板。
- 视觉方向：使用深青绿色玻璃面板、Apple Watch 风格设备环、同步光束和心率弧线动画，保持糖前期健康管理的可信感，避免游戏化或纯炫技。
- 信息结构：先展示 Apple Watch、CGM 传感器、体脂秤与血压计三类设备；Apple Watch 放在优先接入位，CGM 标记为预留接口，体脂秤/血压计标记为可手动同步。
- 工程边界：当前为前端演示状态，接入/取消接入记录在 localStorage；真实授权链路后续再接 Apple Health、CGM 或手动录入 API。
- 可访问性：浮窗按钮使用 `aria-expanded`，展开面板有 region 语义；持续动画只在用户未开启减少动态效果时播放。

### 2026-07-04 移动端触控交互与可访问性全局优化

- **发现问题**：
  1. 移动端 H5 点击 `<a>` 标签时依然会出现系统默认的蓝色/灰色高亮背景，破坏视觉一致性。
  2. 移除了系统默认高亮后，很多按钮（如 `.primary-button`、`.choice-button` 等）没有任何触控按下态的视觉反馈（`:active`），导致操作体验显得“死板”。
  3. 部分按钮和链接的热区较小（如 `.back-button` 为 `36px`，`.text-link` 为 `34px`），不符合移动端至少 `44px` 的无障碍触控目标准则。
  4. 部分卡片和设备按钮有 `:hover` 悬停样式，在触控屏上点击会引起“粘滞悬停”（Sticky Hover），体验不够平滑。
- **决策**：进行全局触控交互规整，让 H5 应用的按压、悬停和点击面积达到原生 App 级品质。
- **实施**：
  - **高亮屏蔽**：将 `-webkit-tap-highlight-color: transparent` 作用范围扩大至 `button` 和 `a` 标签，全面解决默认触控阴影。
  - **触控面积（Touch Target）**：将 `.primary-button` 等主流按钮、`.back-button`、`.ghost-button` 以及 `.text-link` 的 `min-height` 统一提升到 `44px`（或通过 `padding` 扩大热区），避免误触。
  - **按下反馈（Active State）**：为所有核心交互按钮（`.primary-button`、`.tab-button`、`.choice-button`、`.back-button` 等）注入 `:active` 缩放或淡入淡出动画，为 `.action-card` 加上微量按压反馈。
  - **粘滞悬停优化（Hover Query）**：将 `.action-card:hover`、`.device-row-new:hover`、`.device-close:hover` 等悬停动画包裹在 `@media (hover: hover)` 媒体查询中，避免在手机端产生点击后样式粘滞的问题。
