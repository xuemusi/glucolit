# GLUCOLIT Design System

GLUCOLIT is a mobile-first H5 product for prediabetes lifestyle intervention. The interface should feel credible, calm, medical-adjacent, and action-oriented, not decorative or social-media-like.

## Product Personality

- Trustworthy: clear hierarchy, restrained visual language, strong compliance boundaries.
- Human: warm microcopy, no fear-based health scoring, no clinical coldness.
- Action-first: every screen should guide the user to one small behavior.
- Demo-ready: first screen must communicate value in under 5 seconds.

## Layout

- Target viewport: 390 x 844 mobile.
- Desktop should show a centered mobile app shell.
- Use compact dashboard density, not landing-page hero composition.
- No nested cards. Cards are only for repeated items or tool panels.
- Keep bottom navigation fixed with five tabs: 首页, AI, 行动, 陪伴, 科普.
- Use clear full-width sections inside the phone shell.

## Visual Style

- Background: off-white with subtle green tint.
- Primary color: deep metabolic teal.
- Secondary colors: mint green for progress, warm amber for attention, coral only for soft risk hints.
- Avoid heavy gradients, purple-blue, beige-only palettes, and decorative blobs.
- Use 8px radius for cards and controls.
- Use thin borders and soft shadows only on the phone container or primary dashboard surfaces.

## Typography

- Font: Inter or system sans-serif.
- No negative letter spacing.
- Headers should be compact and confident.
- Body text must be readable on mobile and never overlap.

## Components

- Status pill: `需关注`, `已响应`, `今日已完成`.
- Metric cards: concise label, large value, small unit.
- Risk cards: observation + why it matters + action link.
- Tool tabs: report, meal, label.
- Action checklist: five categories with clear done/confirmed/todo states.
- Compliance notice: small but visible near AI outputs.
- Phone companion: calm call UI with waveform, not a flashy call-center look.

## Medical Copy Rules

Use:

- 值得关注
- 建议复查
- 建议咨询医生
- 更适合
- 需控制
- 建议替换
- 今天先完成一个低风险小行动

Do not use:

- 诊断为
- 治愈
- 逆转
- 保证降低血糖
- 不能吃
- 必须吃
- 建议吃药
- 停药

## Screen Direction

Home:

- Top should show GLUCOLIT identity, date, and current state.
- Highlight one "today" narrative: meal spike + sleep + missed check-ins.
- Trend chart should be simple and credible.

AI Tools:

- The upload/sample area should look like a structured lab/product scanner.
- Results must look reviewed and explainable.

Actions:

- Checklist should be the most visually satisfying screen.
- Done state should visibly change the surface.

Companion:

- User-initiated call style.
- One friendly action confirmation.

Content:

- Quiet article and case list.
- No social feed noise.
